# Machine CAD assets

This directory contains one deterministic `.step` asset for every machine that can appear in the layout: the five default factory workcells and every drag-in catalog machine.

The files are generated by `node scripts/generate-machine-assets.mjs`. They are intentionally conservative STEP AP242 proxy assets containing machine identity, footprint, source metadata, and deterministic render-geometry hints. They are used as the app-owned asset set for render scene generation when redistributable OEM CAD is not bundled with the project.

To replace a proxy with an OEM or detailed in-house model, keep the same filename (`<machine-id>.step`) so layouts and render manifests remain deterministic.
