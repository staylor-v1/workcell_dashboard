# Machine CAD assets

This directory contains one deterministic `.step` asset for every machine that can appear in the layout: the five default factory workcells and every drag-in catalog machine.

The files are generated by `node scripts/generate-machine-assets.mjs`. They are conservative STEP AP242 proxy assets containing machine identity, documented footprint dimensions, source metadata, CAD-source search notes, and deterministic render-geometry hints. They are used as the app-owned asset set for render scene generation when redistributable OEM CAD is not bundled with the project.

Catalog machine footprints are keyed to the best available product documentation found during research. When an OEM or third-party CAD model was found but was gated, license-unclear, or caveated as potentially outdated, the corresponding STEP file records that CAD source in metadata and keeps the app-owned proxy geometry scaled to the documented footprint.

To replace a proxy with an OEM or detailed in-house model, keep the same filename (`<machine-id>.step`) so layouts and render manifests remain deterministic.
